# Azure_dev_learn
This I my full learning journey for Azure devOps.
Naming convention for Azure Resources: resourcename-environment-region-sequencenumber
Resource group: It is just like a folder, inside this resource group we can launch multiple reosurces.
Resources: Resources wil be launched in the service provider region depending on the application.
Region: In India there are multiple regions, central, east, west - it just represents a geo-graphical location globally.
Datacenter: Each region has a multiple data centers which is known zone, when we launch a resources we can choose any one of the data center.
Why multiple data centers?: Because cloud gives high available services
Availability Zone: In a region mutiple data centers will be there which is known Zone.
